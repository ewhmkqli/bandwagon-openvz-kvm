## 搬瓦工 OpenVZ 是什么，为什么下架了？

OpenVZ（简称 OVZ 或 VZ）是一种操作系统级虚拟化技术。说人话就是：多个 VPS 共用同一个宿主机内核，资源共享程度更高，但也因此受限更多——不能运行定制内核、不能装 BBR 加速、很多依赖内核的软件用不了。

早年搬瓦工就是靠 OpenVZ 走量的，3.99、5.99、19.99 美元的年付方案吸引了大批用户。OVZ 成本低，价格压得下来，买的人多。

但问题也摆在那里：超售容易、稳定性一般、能跑的软件受限。搬瓦工后来陆续把产品线往 KVM 迁移，等到 2018 年 11 月，直接把所有 OpenVZ 套餐下架，已购的 OVZ VPS 也宣布不允许续费，给了两周数据迁移时间。

从那以后，搬瓦工就彻底是 KVM 的天下了。

## KVM 和 OpenVZ 的实际差距，用过就知道

很多人纠结这个问题，我直接列几个最关键的点：

**内核自由度**：KVM 是完整虚拟化，每台 VPS 有自己独立的内核。想跑 BBR 内核加速？直接选带 BBR 的系统镜像就能用，不用折腾。OVZ 因为共享宿主机内核，这个完全做不到。

**资源隔离**：KVM 下 CPU、内存、硬盘都是真正隔离分配的，旁边机器跑满 CPU 不会影响你。OVZ 超售严重的时候，邻居拉高负载你也跟着卡。

**软件兼容性**：很多工具、Docker、内核级程序在 OVZ 下要么跑不起来要么要改配置。KVM 环境下基本没这个限制。

**迁移灵活性**：搬瓦工 KVM 套餐可以在 KiwiVM 控制面板里一键切换机房，OVZ 当年是无法迁入 CN2 机房的。

搬瓦工做了一件挺少见的事：把 KVM 卖到和当年 OVZ 同等的价格。不涨价，直接升架构。所以现在入手，比当年的 OVZ 用户实际上占便宜了。

## 从 OpenVZ 迁移到 KVM，数据怎么搬？

手里还有 OVZ 老机器、数据没来得及搬的朋友——官方客服确认过，OVZ 和 KVM 之间没法通过快照迁移，技术上不兼容，两种虚拟化架构差异太大。

只能手动搬数据：

1. 在 OVZ 机器上打包需要的文件（`tar -czf backup.tar.gz /var/www /etc/nginx` 之类）
2. 购买一台新的 KVM 方案
3. 用 scp 把文件传过去解压
4. 重新配置服务

IP 地址会变。这没办法，两台不同的机器。

如果 OVZ 机器已经到期，系统一般还会保留几天，抢时间提工单申请临时开机把数据拿出来。超过这个窗口期就彻底没了。
